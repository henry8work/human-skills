## 0. Doctrine

The greatest source of learning in design isn't theory — it's **carefully curated and analyzed reference**. A designer who looks at 50 strong references per week evolves more than a designer who reads 5 theoretical books per year. The eye is formed by exposure.

The difference between a brand with strong visual identity and one without is frequently a difference in **reference library**. A brand that keeps a moodboard alive, updated, analyzed has a repertoire to discuss visual decisions concretely. A brand without a library debates each decision on the basis of "hunches".

Every decision in this document starts from a premise: **reference without analysis is decoration**. Having a folder full of Pinterest screenshots isn't a library — it's a collection. A library requires analysis: what does this piece teach? Why does it work? What pattern repeats across multiple refs?

The function of this file is to make the reference library **a living organism** that feeds every creative decision of the brand. Not frozen. Not decorative. Operational.

## 2. The 12 founding principles

### Principle 1 — Curation is analysis, not collection

A folder full of screenshots without analysis is visual clutter. A useful library requires **analysis per reference**:

- What does this piece teach?
- Why does it work?
- What technical pattern is visible?
- What could be applied to our brand?

Without analysis, the reference doesn't become repertoire — it stays as static decoration.

### Principle 2 — Quantity doesn't beat quality

50 careful + analyzed references beat 500 random screenshots. A brand that tries to have a "moodboard of 200 images" usually can't navigate its own library when needed.

Healthy standard: 30-80 approved refs per brand, divided into 5-7 categories, with 1-2 lines of analysis per ref.

### Principle 3 — Reference needs context

A loose image on Pinterest loses 80% of its value. A useful ref has:
- Source (where it came from)
- Author/photographer/designer (who made it)
- Date (when it was made)
- Context (what it was for)
- Analysis (what it teaches)

Without context, ref becomes generic inspiration.

### Principle 4 — Anti-reference is as important as reference

Visual identity is built as much by inclusion as by exclusion. A library of "what we don't want to look like" guides as much as a library of "what we want to look like".

Every brand should have, at minimum:
- 30-50 positive references (inclusion)
- 10-20 anti-references (exclusion)

### Principle 5 — Real vision > textual description

Visual reference stored **as image** is worth more than as textual description. AI that reads the library via real vision (Claude Read, GPT-4V) extracts visual patterns that text description misses.

Implication: a digital library that has only links without stored images is fragile. Images need to be in the `materiais/3-referencias-visuais/` folder locally, or in a synced folder (Drive, Notion) accessible.

### Principle 6 — Grid as composite reference

Feed screenshot (Behance, Instagram, Pinterest) with multiple slides in one image = grid. Each region of the grid is an independent ref.

Useful ref: **one image in a good grid can be worth 9 separate refs**. Don't force quantity — attach what actually represents.

### Principle 7 — Continuous curation > big bang

A useful library isn't assembled once at setup and forgotten. It's fed **continuously**:
- 5-15 new refs per week (R1 Brand Scout)
- Weekly human curation (20 min)
- Quarterly refresh (remove aged refs)
- Annual snapshot in Drive

Without continuous feeding, the library ages in 18 months.

### Principle 8 — Diversity of sources avoids endogamy

A brand that only consumes reference from Pinterest ends up looking like Pinterest. A brand that diversifies sources (Are.na + Behance + physical magazines + books + gallery + architecture + cinema) builds a unique repertoire.

Recommended sources:
- 30% Are.na (curatorial)
- 20% Pinterest (popular but filter)
- 15% physical magazines/books
- 15% institutional sites (Pentagram, Wolff Olins, MoMA)
- 10% cinema/theater/exhibitions
- 10% direct competitors (calibration)

### Principle 9 — Pattern detection is where value lives

5 individually good references don't teach as much as 5 references analyzed together to extract a recurring pattern.

Pattern analysis:
- What typography repeats?
- What kind of light?
- What palette?
- What kind of composition?
- What detail signature appears in multiple?

Recurring pattern = implicit system. Documenting = explicit system.

### Principle 10 — Refresh avoids stagnation

Aesthetics change. Trends age. A 2020 library doesn't serve 2026. Structured refresh:

- **Quarterly:** remove clearly aged refs
- **Semi-annually:** complete snapshot of the library (historical archive)
- **Annually:** critical review — does the library still represent the current brand?

A brand that doesn't refresh ends up trapped in an aged aesthetic without realizing it.

### Principle 11 — Thematic package > single library

When starting a specific project (campaign, launch, redesign), create a focused **thematic package**:

- 15-30 specific refs for the project
- Specific anti-refs
- Direction notes for the project

The thematic package takes priority over the general library during the project. Temporary focus.

### Principle 12 — Library is alive, not a monument

Worst library anti-pattern: creating an elaborate system, abandoning it in 6 months. A useful library is **proportional to frequency of use**.

Principle: a library grows naturally when consulted. If no one consults it, it's dead — not worth keeping.

Small brand: lean library of 30-50 refs, quarterly refresh.
Large brand: broad library with sub-categories, weekly curation.

## 3. Library structure — 6 operational sections

### 3.1 — Section A: Approved visual refs (human-curated)

Images that define the brand's visual system. Real vision analyzes them when R2 will generate a new piece.

Each approved reference has:
- The image (attached)
- Description (1-2 lines explaining what makes it work)
- Category (cover, inner slide, web hero, social post, other)
- Tags (keywords that help R2 search)
- Observed pattern (what visible technique)

**Ideal quantity:**
- Carousel covers: 5-15
- Inner slides: 10-20
- Hero shots: 5-10
- Editorial posts: 10-20
- Mood boards (grids): 3-8

Total: ~50-100 visual refs is a healthy library.

### 3.2 — Section B: Approved editorial refs (model texts)

Texts that **represent the right tone**. Function as living calibration of the `🗣️ Voice & Tone`.

Each reference has:
- The text (paste whole or link + summary)
- Type (caption, long post, email, headline, CTA, etc.)
- Why it works (1-2 lines analyzing)
- Origin (own approved / other admired brand)

**Ideal quantity:**
- Captions: 10-20
- Long posts: 5-10
- Emails: 5-10
- Headlines: 10-30 (inspiration bank)
- CTAs: 5-15

### 3.3 — Section C: Behavioral refs

Screenshots of well-done DMs, model crisis communications, exemplary threads.

Each reference:
- Print/capture (anonymize sensitive data)
- Context (what happened)
- Why it works (analysis)

### 3.4 — Section D: Anti-references

Explicitly listed — the more specific, the better the result.

**Visual anti-references:**
- We don't want to look like generic Apple minimalism
- We don't want to look like tech AI purple gradient
- We don't want to look like blue SaaS dashboard
- We don't want to look like basic Bauhaus
- We don't want to look like torn paper texture

**Editorial anti-references:**
- We don't want to sound like a marketing guru
- We don't want to sound like a traditional agency
- We don't want to sound like a tech bro
- We don't want to sound like a motivational coach

**Behavioral anti-references:**
- We don't want to act like a brand that deletes critical comments
- We don't want to act like a brand that gives 70% BF and charges inflated prices in the other 11 months
- We don't want to act like a brand that pretends to be a person in DMs

### 3.5 — Section E: Pending (R1 feeds, human curates)

R1 (Brand Scout) automatically collects relevant inspirations for the brand and files them here.

**How R1 collects:**

| Source | What it looks for |
|---|---|
| Pinterest (boards related to brand_aesthetic_anchor) | Images that match the aesthetic |
| Are.na (related channels) | Curated editorial and visual refs |
| Behance (related categories) | Complete projects |
| Dribbble (related tags) | UI / web / brand identity |
| Instagram (handles defined as reference) | New posts from anchor brands |
| Designspiration / Cargo / Fwa | Editorial-leaning refs |

**Collection criteria:**
- Minimally matches brand_aesthetic_anchor
- Doesn't violate anti-references (Section D)
- Has minimum 1 clearly aligned dimension (palette, typography, composition, tone)

R1 can bring 5-15 pending per week. Weekly curation: ~20min.

**Decision for each pending:**
- ✅ Approve → moves to Section A/B/C
- 🗂️ Archive (good, not for now) → "Future bank"
- ❌ Reject → delete

### 3.6 — Section F: Competitive map

Monitored competitors — not to copy, to **consciously differentiate**.

| Competitor | Handle | Strengths | Where we differ |
|---|---|---|---|
| [Name 1] | @ | [...] | [...] |

**Periodic snapshots:**
R1 creates a monthly snapshot of each competitor (5-10 most recent posts, evolving visual pattern, tonal pivot).

**Quarterly comparative analysis:**
Every 90 days, R2 generates a structured comparative analysis.

## 4. Real vision — image analysis via AI

When R2 will use the library to feed the prompt:

1. `ls materiais/3-referencias-visuais/` lists the images
2. `Read` on each image (Claude native vision)
3. Structured analysis per image:
   - Visible typography
   - Dominant palette + accent
   - Predominant composition
   - Lighting (if photo)
   - Detail signature
   - Mood

4. Compilation in **decoded visual briefing** (300-500 words):

```
VISUAL BRIEF (decoded from references):

Color palette:
- Primary: [hex] ([description])
- Background dark: [hex]
- Background light: [hex]
- Accent secondary: [observed in references]

Typography style:
- Headlines: bold condensed sans-serif, weight 900, very tight kerning, uppercase, similar in feel to [typographic reference]
- Body: clean humanist sans-serif, weight 400-500, medium contrast, similar to [reference]
- Tag: monospaced or geometric mono, weight 600, uppercase

Mood and atmosphere:
- [extracted from approved refs — 2-3 sentences]

Composition style:
- [extracted from approved refs — 2-3 sentences]

Texture and finish:
- [extracted from approved refs — 1-2 sentences]

Detail signature / footer:
- [observed in multiple refs as recurring element]

What to AVOID (from anti-references):
- [list from Section D]
```

This briefing enters as a **fixed block in all prompts** (cover and inner slides) of that run.

## 5. Grid detection and decomposition

When someone attaches an image that is a **grid** (feed screenshot, Behance page, moodboard with multiple slides), R2 detects and decomposes.

**Grid signs:**
- Spatial regularity (clear lines/columns)
- Repeated borders
- Square or rectangular format subdivided
- Multiple similar rectangles side by side

**How to decompose:**
- 3×3 grid = 9 specimens
- Feed screenshot = 6+ specimens
- Behance screenshot = N specimens per layout

**Analysis:**
Each specimen of the grid is observed as an independent ref. Recurring patterns between specimens = markers of consistency (visual system).

**Practical implication:**
**a single image in a good grid can be enough.** Don't force quantity — attach what actually represents.

## 6. Anti-references — as important as references

### 6.1 — Why anti-ref is strategic

Identity is built by **inclusion + exclusion**. Without clear exclusion, brand dilutes in 5 years.

Generative AI, specifically, gravitates toward the generic by design. Without explicit anti-prompt, it returns the average of training. A documented anti-reference becomes `avoid:` in the image_brief.

### 6.2 — Specificity of the anti-ref

Vague anti-ref ("bad design") filters nothing. Specific anti-ref filters everything.

✅ Specific:
- "We don't want to look like a 2018-2022 B2B SaaS brand with purple-blue gradient + stock photo of diverse smiling team + heading 'The complete solution for...'"
- "We don't want to sound like a Brazilian motivational coach with 'believe in your potential' + fire emoji + inspirational audio in the background"

❌ Vague:
- "Modern design"
- "Institutional tone"

### 6.3 — How to capture anti-ref

Direct question in the brief: "Cite 3 brands/styles/pieces you definitely do NOT want to look like. Why?"

Each anti-ref comes with:
- Print/example (when possible)
- Reason to avoid
- Category (visual, verbal, behavioral)

## 7. Continuous curation via R1 (Brand Scout)

### 7.1 — Collection frequency

R1 runs 3x per week (default Mon, Wed, Fri at 10am). Each execution:
- Collects 5-15 candidates
- Adds to Pending (Section E)
- Tags marks and source

### 7.2 — Human curation

Every Wednesday (or another weekly ritual), the human curator (designer, founder, brand person) spends 20-30 min:
- Reviews pendings
- Approves / archives / rejects each
- Adds analysis to approved ones
- Moves to corresponding Section A/B/C

### 7.3 — Quarterly refresh

Every 90 days:
- Review Section A entire: still represents the system?
- Remove clearly aged refs
- Add new refs if system has evolved
- Re-organize categories if necessary

### 7.4 — Annual snapshot

Once a year:
- Complete export of the library to Drive
- Archived version of the library
- Analysis: how has the library changed in 12 months?

The snapshot serves as **institutional memory**. The library from 5 years ago tells the visual story of the brand.

## 8. GOOD vs BAD curation (canonical cases)

### 8.1 — GOOD curation (study)

**Exemplary cases:**

- **swissmiss** (swissmiss.com) — Tina Roth Eisenberg curates since 2005, daily. Unmistakable editorial consistency.
- **It's Nice That** (itsnicethat.com) — premium editorial curation in contemporary design.
- **Brand New** (UnderConsideration) — critical analysis of visual identities. Each analysis is a case study.
- **swissted** (swissted.com) — Mike Joyce reimagines punk posters in Swiss aesthetic. Authorial curation in a project.
- **Pentagram Papers** (50+ editions since 1975) — curation of specific topics by studio designers. Authorial curation model.
- **Stack Magazines** (stackmagazines.com) — monthly delivery of curated independent magazine. Curation as service.

**Common characteristics:**
- Consistent frequency (daily, weekly, monthly)
- Authorial voice (identifiable curator)
- Analysis per item (not just compilation)
- Source diversity (not just one platform)
- Explicit criterion (what enters and why)

### 8.2 — BAD curation (avoid)

**Saturated anti-patterns:**

- **Pinterest folder without analysis** — collection, not library
- **Moodboard of 500 images** — no one can navigate
- **Abandoned library** — zero updates for 18 months
- **Curation without voice** — generic collection without angle
- **Everything from one source** — visual endogamy
- **No anti-ref** — only inclusion, no exclusion

**Symptoms in medium-sized Brazilian brands:**
- Designer opens Pinterest for "inspiration" each project
- No documented brand library
- Each piece is born from scratch (without accumulated repertoire)
- New team has nowhere to study the brand's aesthetic

## 9. Library that WORKS vs DOESN'T WORK (at scale)

### 9.1 — WORKS

- **Lean library + analyzed** (30-80 refs with analysis)
- **Quarterly refresh** (continuous feeding)
- **Accessible real vision** (stored images, not just links)
- **Documented anti-refs**
- **Thematic packages** when starting a project
- **Weekly curation** (incorporated ritual)
- **R1 feeding pendings**

### 9.2 — DOESN'T WORK

- **Huge abandoned library** (500 refs without analysis, without refresh)
- **Random refresh** (update "when there's time")
- **Only links without images** (fragile — links die)
- **No anti-refs** (only inclusion)
- **No thematic packages** (each project starts from scratch)
- **Sporadic curation** (no ritual)
- **Criterion-less collection** (anything enters)

## 10. Thematic packages — curation by project

When starting a specific project (campaign, launch, redesign), creating a **focused package** accelerates creative direction.

### 10.1 — Package structure

```
Package: [Project name] — Q[N] [Year]

Applicable visual refs (from Section A):
- A-04, A-12, A-23, A-31

Applicable editorial refs:
- B-08, B-15

Additional specific refs (only for this package):
[attach 5-10 specific refs]

Anti-references relevant for this campaign:
- "Nothing that looks like a Black Friday campaign"
- "No fabricated urgency"
- "No influencer posing"

Direction notes for the project:
- [observation 1]
- [observation 2]

Status: ACTIVE / ARCHIVED
Start date: [date]
End date: [date]
```

### 10.2 — How R2 uses the package

When there's an "Active thematic package", R2 prioritizes it over the entire Library during the project. Temporary focus.

The package serves as a **curated brief for the specific project** — concentrates creative direction.

### 10.3 — Package archiving

When the project ends:
- Status: ARCHIVED
- Keep in the Library for history
- Specific refs that survived = candidates to enter the permanent Section A

## 11. Refresh and archiving

### 11.1 — Tasks by frequency

| Task | Frequency |
|---|---|
| Triage pendings | Weekly |
| Re-curate Section A (approved visual) | Quarterly |
| Re-curate Section B (approved editorial) | Quarterly |
| Add new cases to Section C | When a good case happens |
| Re-curate Section D (anti-refs) | Quarterly |
| Update Section F (competitive map) | Monthly |
| Complete snapshot in Drive | Quarterly |
| Complete strategic review | Annually |

### 11.2 — Future bank

Refs that are good but don't fit at the moment:
- Aesthetic too bold when the brand is in a conservative phase
- Reference whose theme is not on the agenda
- Emerging trend that may become relevant

The bank keeps for future review. At each quarterly refresh, the bank is re-evaluated.

### 11.3 — Visual evolution history

Every 6 months, visual snap of the brand for history. Visual identity timeline:

| Period | Predominant aesthetic | Milestones |
|---|---|---|

Worth as a timeline. Useful for understanding "how we got here" and "how we can go back / evolve".

## 13. Operational checklist

### Before generating a piece that uses the library

1. `ls materiais/3-referencias-visuais/` — list available refs
2. `Read` on each image (real vision)
3. Detect grids and decompose
4. Identify recurring patterns
5. Compile decoded visual brief (300-500 words)
6. Inject into the prompt as a fixed block

### Weekly curation of pendings

1. Open Section E (Pendings collected by R1)
2. For each pending: review R1's analysis
3. Apply approval criterion:
   - Matches brand_aesthetic_anchor?
   - Doesn't violate anti-refs?
   - Has clearly aligned dimension?
4. Approve / Archive / Reject
5. Move approved ones to corresponding Section A/B/C

### Quarterly refresh

1. Review Section A: still represents the system?
2. Remove aged refs
3. Add new refs if system has evolved
4. Re-organize categories if necessary
5. Complete snapshot in Drive
