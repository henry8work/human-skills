# resultado/ — final DNA delivery

This folder receives the project's official deliverables:

- `DNA.md` — operational source for AIs and agents;
- `DNA.pdf` — **single final PDF**, editorial document laid out as a 16:9 presentation, detailed and presentable;
- `DNA.html` — auxiliary file used to inspect the PDF when needed.

The PDF must use the materials from `referencias/` as visual and strategic foundation: concept, context, central idea, main tension, brand territory, interpreted references, reference → meaning → rule matrix, logo when it exists, palette, typography, images, texts, tone of voice, anti-references, analyzed mockups and usage instructions.

Do not generate separate PDFs for palette, typography, applications, brand book or attachments. Everything goes into the `DNA.pdf`. `.md`, `.html` files, images, logs and inventories may exist as backstage material, but the consultable and presentable material is a single PDF.

The `DNA.pdf` must feel like an editorial presentation about the brand, with pages in slide rhythm. Simple, direct, human language, without unnecessary jargon and without em-dashes. Depth comes from the interpretation of references and the number of pages needed, not from squeezing text.

Whenever possible, the PDF itself should absorb the analyzed visual identity: logo, colors, typography, margins, breathing room, visual density, grid, empty spaces and composition rhythm. If there is a logo with sufficient quality, it should appear in the document. If there isn't one, the document continues with the visual signals available.

It is not acceptable for the PDF to look like printed markdown. The palette must appear as real color blocks, images must be laid out as a visual study, and the read must look like a brand book, magazine or advanced deck. If there's a site with many images, the gallery and analysis must reflect that abundance.

When images sent by the user contain lists, tables or structure examples, they must be read as content. The aesthetic of these images does not automatically become the visual direction of the PDF. The DNA must extract what they ask for: categories, questions, examples, criteria and depth.

Matrices expected when there is sufficient material:

- References: brands, directors/photographers, design/editorial, cinema and culture;
- Tone of voice: general tone, rhythm, vocabulary, what to avoid, humor and positioning;
- Visual: palette, aesthetics, composition, light, texture and references.
