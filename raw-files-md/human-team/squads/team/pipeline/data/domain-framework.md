# Domain Framework — Time Criativo

> Metodologia operacional unificada que os 10 agentes seguem. Define o **passo-a-passo** que transforma uma ideia bruta em conteúdo publicável.

## Fluxo macro

```
Ideia bruta
   ↓
[Brief] → Planner → Scout → Creative Director → (✋ aprovar conceito)
   → Scriptwriter → (✋ aprovar roteiro)
   → Art Director → Storyboarder → (✋ aprovar storyboard)
   → Producer → Editor → (✋ aprovar final)
   → Social Manager → Content Multiplier
   ↓
Conteúdo publicado + N derivados em outros formatos
```

## Etapas detalhadas

### 1. Brief (step-00)
**Entrada**: ideia bruta do usuário. **Saída**: brief estruturado com objetivo, público, formato pretendido, tom, restrições.

Sem brief sólido, todo o resto vira ruído. Quando o usuário descreve algo vago, o sistema faz perguntas direcionadas até cobrir: quem é a audiência, qual é a ação esperada, qual é o tom, qual é o formato preferido, quais são as restrições/compliance.

### 2. Planning (Planner)
Traduz o brief em **plano executável**: lista de entregáveis, ordem de produção, dependências, definition of done de cada peça, prazos relativos. Aplica backward planning a partir da data de publicação.

### 3. Research (Scout)
Investiga **3 frentes**: (a) tendências da categoria, (b) audiência (linguagem, dor, desejo), (c) referências de execução (o que já funcionou em peças similares). Sai com um dossiê de no mínimo 5 fontes citadas.

### 4. Concept (Creative Director)
Sintetiza brief + dossiê em **big idea**: uma frase de conceito que cabe num post-it e dirige todas as decisões seguintes. Inclui ângulo, gancho e justificativa do porquê essa ideia vence as alternativas.

### 5. Script (Scriptwriter)
Materializa a big idea em **copy/headlines** com hook, desenvolvimento e CTA. Estrutura varia por formato (KV, carrossel, anúncio, e-mail, etc.). Aplica uma das fórmulas testadas: Hook-Story-Offer, AIDA, PAS, ou Save-the-Cat para narrativas longas.

### 6. Art Direction (Art Director)
Define o **sistema visual**: paleta, tipografia, mood, referências de iluminação, regras de composição. Entrega um art bible curto que o storyboarder e o editor seguem.

### 7. Storyboard (Storyboarder)
Quebra a copy em **quadros visuais**: cada beat vira uma peça/quadro com descrição de formato, plano/layout, composição, ação visual e texto aplicado. Para conteúdo estático, define grid e hierarquia visual de cada slide/peça.

### 8. Production (Producer)
Lista todos os **assets necessários** (KV, fotos, ilustrações, logos, referências, fontes) e o **caminho de obtenção**: gerar, capturar/produzir, comprar em banco, buscar no acervo. Cria a folha de produção com responsável e prazo de cada item.

### 9. Editing (Editor)
Finaliza o **material estático** seguindo o storyboard e a art direction. Audita texto aplicado, proporção, safe zones, legibilidade, logo, KV e organização dos arquivos finais.

### 10. Distribution (Social Manager)
Adapta o conteúdo para cada canal (legendas, variações estáticas, hashtags, primeira frase) e **publica** seguindo o cronograma definido pelo Planner. Para ações irreversíveis (publicação), pausa para aprovação humana.

### 11. Multiplication (Content Multiplier)
Pega a peça principal e **deriva N variações** em outros formatos estáticos/editoriais: carrossel, post, thread, newsletter, stories, anúncio e e-mail. Cada derivada respeita as melhores práticas do formato destino.

## Definition of Done (DoD) global

Uma execução só é considerada concluída quando:
- [ ] Brief original anexado ao output
- [ ] Big idea explicitada
- [ ] Roteiro/copy final versionado
- [ ] Art bible aplicado de forma consistente
- [ ] KV/peça principal entregue com versões por canal
- [ ] Publicação confirmada (com datas/horários e URLs)
- [ ] Pelo menos 3 derivadas em outros formatos
- [ ] Memória da squad atualizada com aprendizados
