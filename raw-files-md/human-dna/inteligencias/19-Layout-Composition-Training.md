# 19 — Advanced Layout and Composition Training

This module consolidates the best learnings on layout, design, composition, carousel, presentation, product photography, motion and storyboard from the Human ecosystem.

It exists to prevent the `DNA.pdf` from being redesigned by trial and error. Before rendering any DNA, the Maestro must use these rules as a practical repertoire of art direction.

## Central principle

The DNA PDF is an editorial brand presentation. Every page must have:

- a clear central idea;
- an object of visual or textual proof;
- its own composition;
- a reading hierarchy;
- a reason to exist.

If the page doesn't have idea, proof and composition, it's not a page. It's leftover.

## 1. Design system before the page

Before laying out, define the document's visual system:

- dominant palette, secondary, accent, background and text;
- title font, body font and metadata font;
- size scale for cover, section, title, body, caption and big number;
- fixed margins;
- base grid;
- style of lines, cards, tables, icons, swatches and visual boards;
- photography treatment;
- level of rounding, shadow and frame, if any.

No page can invent style from scratch. Layout variation isn't a mess. It's using the same system in different compositions.

## 2. Three reading levels

Every page must have exactly three clear levels:

1. **Anchor:** what the eye sees first. Can be title, big number, image, logo, dominant swatch or short phrase.
2. **Context:** what explains the anchor. Can be short paragraph, list, matrix, caption or analysis.
3. **Metadata:** what organizes without competing. Can be section, page, reference origin, category or short note.

If two elements fight for first place, the hierarchy failed. If everything has the same size, the page looks like a poor template.

## 3. Claim + proof

Each slide needs to say something, not just name a theme.

Bad:

- `Brand territory`
- `Colors`
- `Photography`

Good:

- `The brand builds desire through low contrast, isolated product and visual silence`
- `Red appears as a signature, not as a dominant background`
- `Photography sells ritual, not just product`

After the assertion, the page must show the proof: photo, crop, swatch, matrix, real sentence, post pattern, image sequence, found typography or comparison.

## 3.1 Technical term needs to become usage

Technical content can never appear as a riddle. If the DNA cites WCAG, contrast 4.5:1, RGB, HEX, baseline, scale, grid, aspect ratio, safezone, DPI, crop, CSS, font variable, fallback or any backstage expression, the page must translate that immediately.

Every technical term must answer:

- what it means in simple language;
- how to apply on the brand;
- approved example;
- common error.

Examples:

- **WCAG / contrast:** means the text needs enough strength to be read without effort. Apply using dark text on light background or light text on dark background. Approved example: `ink` on `cream`. Common error: colored text on colored background just because the two colors belong to the palette.
- **HEX / RGB:** means the exact code of the color. Apply showing the swatch next to the name and the usage. Approved example: square `#F4EFE5` as editorial background. Common error: listing ten codes without saying which is background, text, CTA or accent.
- **Baseline / grid:** means aligning elements so the page looks intentional. Apply keeping titles, images and captions locked to a structure. Common error: leaving icon, number or text block loose with no relation to the rest.
- **Aspect ratio / proportion:** means the image's format. Apply preserving face, product, packaging, logo and gesture. Common error: cropping a photo to fit a pretty rectangle, but destroying the visual evidence.

Final rule: technical term only enters the PDF if it helps the reader decide better. If it doesn't become practical action, it stays out of the presentable document.

## 4. Mental contact sheet

Before delivering, the Maestro must imagine the PDF in thumbnails.

In thumbnail, the document must show:

- coherent visual system;
- pages with different rhythms;
- alternation between text, matrix, image, big data and synthesis;
- visual repertoire appearing beyond the start;
- absence of empty or almost-empty pages;
- no sequence of three pages with the same structure.

If in thumbnail everything looks the same, the document lacks direction. If in thumbnail it looks random, there's no system.

## 5. Tonal rhythm

The PDF must respect the brand's tonal register.

- Light brand asks for a light document, with elegant contrast.
- Dark brand asks for a dark or mixed document, but legible.
- Colorful brand asks for color with function, not carnival.
- Minimalist brand asks for breathing, but not abandonment.
- Aggressive brand can have hard contrast, typographic weight, sharp cuts and more frontal visual language.

Don't darken a light brand just to look premium. Don't lighten a dense brand just to look clean. The tone comes from the evidence.

## 6. Right density per page

Light pages support more text. Dark pages tire faster. Pages with image need less text. Matrix pages need breathing per row.

Practical rules:

- If there's little text, increase the visual force or group related topics.
- If there's a lot of text, divide into topics, matrix or two columns.
- If a card has overflowing text, the card is wrong.
- If a card has text lost in the corner, the card is wrong.
- If a section yields three weak pages, consolidate into one strong page.

## 6.1 Intelligent pagination

The agent can't paginate like a markdown converter. It needs to paginate like a designer.

Before opening a new page, evaluate:

- do the points belong to the same subject?
- are they short enough for a compact matrix?
- do they fit in two or three columns with breathing?
- can the subtitle become a label inside the same page?
- is the previous page really full?
- would the new page have less than half of the useful area occupied?

If two pages are weak apart and strong together, join them. If a page exists only because a subtitle appeared in the markdown, it's wrong.

Empty chapter divider is almost always waste. Only use a chapter opening when it carries context, narrative shift, hero image, synthesis or visual tension. Otherwise, the title enters together with the first block of the section.

Consolidation models:

- **3 to 6 short topics:** small cards or vertical numbered list.
- **7 to 15 short topics:** compact matrix in three columns.
- **3 to 8 medium topics:** two editorial columns.
- **1 strong idea + 2 supports:** statement slide.
- **Sibling subsections:** group on one page with internal labels.
- **Small subtitle after a few topics:** doesn't break page; becomes internal label and continues the composition.
- **Small tables:** cards or light matrix, never almost-empty page.
- **Table that overflows by 1 or 2 lines:** reduce visual weight, use compact mode or convert into cards. Short continuation is a design error.

Visual criterion: a common page must occupy at least 45% of the useful area. Below that, it needs consolidation, image, visual proof or restructuring.

## 7. Visual references as grid

Moodboards, Instagram screenshots, Behance pages and campaign grids aren't a single image. They can contain many examples within the same image.

When a reference is a grid, the Maestro must mentally decompose:

- how many pieces exist there;
- which have the most strength;
- what hierarchy is repeated;
- what type of image appears;
- how title, caption, texture, color and signature behave;
- what is the rule and what is the exception.

A good reference in grid can teach more than ten loose images. The PDF must reflect that reading.

## 8. Typography locked by function

The document needs to identify and apply typographic roles:

- title;
- subtitle;
- body;
- button or CTA;
- caption;
- metadata;
- big number;
- signature/logo.

It's not enough to say the font is `sans-serif` or show a CSS variable. The user needs to understand which font or approximation to use in each function.

Once chosen, the PDF's typography can't vary randomly. Variation happens by size, weight, case, position and scale, not by changing font without reason.

## 9. Color with function

The palette isn't a list of hexadecimals. It's a usage system.

Each color must have function:

- dominant background;
- surface;
- title;
- text;
- CTA;
- CTA text;
- accent;
- rare color;
- forbidden or dangerous color.

Accent color must appear with intent: keyword, thin line, swatch, detail, marker or data. Don't use accent as a giant block if the brand doesn't do that.

## 10. Image as proof, not decoration

Image enters to teach:

- light;
- shadow;
- pose;
- product;
- scale;
- texture;
- crop;
- surface;
- gesture;
- atmosphere;
- editorial repetition.

Real photo isn't banner, textual screenshot, card, UI, flat mockup or logo. These materials can be analyzed, but don't enter the photographic gallery.

When there are many valid photos, use bento grid, mosaic or editorial boards. Preserve proportion. Don't mutilate product, face, packaging, lettering, logo or gesture.

The page name must not reveal the technique behind it. "Photo bento" is internal language and sounds strange to whoever reads. In the PDF, use human titles: "Photo gallery", "Photos analyzed", "Visual board", "Light study" or "Photographic rhythm".

## 11. Visual continuity

The DNA must look like a single document, not pages assembled by different agents.

Check continuity:

- dominant color per section;
- recurring title position;
- margin size;
- caption style;
- treatment of photos;
- table style;
- distance between blocks;
- role of icons;
- language of dividers.

Variation without continuity looks like collage. Continuity without variation looks like template.

## 12. Photographic direction applied to layout

When the brand has photography, the layout must absorb the photography.

Observe:

- frontal, side, hard, diffuse, low-key or natural light;
- short, long, dramatic or almost absent shadow;
- clean background, real setting, surface, fabric, metal, paper, skin, glass;
- frontal pose, casual gesture, editorial, isolated product, object still;
- camera distance, crop, negative space and focus.

The PDF must use these decisions to assemble boards, crops and analysis pages. Don't throw an image without a read.

## 13. Layers and depth

A visually rich page doesn't need ornament. It needs well-decided layers:

- background;
- surface;
- image or proof;
- title;
- body;
- metadata;
- functional detail.

Functional detail is what helps reading: line, marker, swatch, icon, number, label, light texture or crop. If it doesn't help, it goes out.

## 14. Strong opening

The first pages define the reader's trust. They need to show that the DNA understood the brand.

Weak opening:

- cover with generic text;
- operational metadata;
- "master file";
- empty page with a loose sentence;
- small logo lost.

Strong opening:

- brand name with presence;
- useful synthesis sentence;
- logo when extracted;
- real palette;
- a hero image or decisive visual detail;
- hint of the creative territory.

## 15. Table and matrix pages

A good table doesn't look like a spreadsheet.

It needs to have:

- clear header;
- columns with width thought through;
- rows with breathing;
- subtle contrast between rows;
- enough internal padding for the text to breathe;
- row height calculated by the longest content of the row;
- separators below the text, never crossing the reading;
- text broken correctly;
- swatches when talking about color;
- hierarchy between category, question, evidence and rule.

If the border appears above the text, touches the text or creates visual noise, the matrix failed. First fix the cell layout; then evaluate content. If the table has little content, turn into cards or list. If it has too much content, divide by theme.

## 16. Pages with big data

A big number needs to stay glued to its label.

Correct example:

```text
48
unique photos analyzed
```

The number, the unit and the explanation form a single visual block. Never separate the number from its caption by a huge space.

## 17. Rendered verification

Before delivering:

- export or view PDF pages;
- look at least at cover, visual gallery, palette, typography, matrix and dense pages;
- look for bad cuts, overflowing text, gaps, broken accents, sentence starting with lowercase, fragment without explanation, layout repetition and poor pages;
- if the error appears in preview, fix before replying.

Layout isn't validated only by code. It's validated by looking.

## 17.1 Warm and didactic writing

The reader has no obligation to know design, branding, typography, photography or development. The document needs to teach without infantilizing.

Each visible topic needs to:

- start with a capital letter;
- have correct punctuation;
- explain the expected action;
- explain the reason for the rule when there's risk of doubt;
- avoid loose fragments that look like internal notes.

Bad example: `Two accents in the same piece.`

Good example: `Avoid using two accent colors on the same piece, because they compete with each other and leave the composition confusing. Choose one highlight color at a time.`

If the sentence looks like an internal note, it's not yet ready for the PDF.

## 18. Absorbed anti-patterns

Never do:

- template example content presented as if it were the brand (names, sentences or colors copied from the template/protocol instead of extracted from the real brand);
- generic renderer visual applied identically to different brands, without deriving color and typography from the analyzed brand;
- inflating the document to 40, 50 or 60+ pages by paginating text instead of consolidating;
- background with generic gradient unrelated to the brand;
- loose icon just to "decorate";
- card inside a card;
- table with fewer than three rows;
- centered text on dense pages;
- repeated image as decoration;
- stock photo as brand proof;
- bullets glued in one line;
- raw technical text for non-technical user;
- sentence starting with lowercase;
- short topic without context, reason or practical guidance;
- font without accent;
- page with only title and a small sentence;
- logo repeated on every page without function;
- color accent on entire sentences;
- three pages in a row with the same macro-structure;
- visual proof without context;
- deep analysis hidden in illegible text.
- decorative side rail;
- "DNA SIGNALS" block;
- generic icons used to fill emptiness;
- loose swatches outside of color, palette, application or chromatic evidence pages.

## 19. Advanced checklist

Before rendering the final PDF, answer:

- What is the document's visual system?
- What are the three reading levels of each critical page?
- Does each page have an assertion and a proof?
- Does the document win the thumbnail test?
- Does the tonal register respect the brand?
- Do real photos appear in proportional volume?
- Were reference grids decomposed?
- Was typography locked by function?
- Do colors appear with function and swatch?
- Does the opening convey brand, not metadata?
- Is there continuity without monotony?
- Is there variation without collage?
- Does any page look like markdown?
- Does any page look like a filled template?
- Was any image cut for convenience?
- Did any text overflow, end up too small, or sit lost inside a large block?

If any important answer is bad, the PDF isn't yet ready.
